# Dialog

Dialogs inform users about a task and can contain critical information, require decisions, or involve multiple tasks. @duskmoon-dev/core provides a complete set of Material Design 3 dialog variants.

## Basic Usage

A basic dialog consists of a backdrop, dialog container, header, body, and actions.

<ComponentShowcase title="Basic Dialog" code={`<div class="dialog-backdrop" id="myDialog">
  <div class="dialog">
    <div class="dialog-header">
      <h2 class="dialog-title">Dialog Title</h2>
      <button class="dialog-close" aria-label="Close dialog">
        <svg class="w-5 h-5" fill="none" stroke="currentColor" viewBox="0 0 24 24">
          <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M6 18L18 6M6 6l12 12" />
        </svg>
      </button>
    </div>
    <div class="dialog-body">
      <p>This is the dialog content. You can add any content here.</p>
    </div>
    <div class="dialog-actions">
      <button class="btn btn-text">Cancel</button>
      <button class="btn btn-primary">Confirm</button>
    </div>
  </div>
</div>`} />

To show the dialog, add the `dialog-backdrop-show` class to the backdrop:

<ComponentShowcase title="Showing Dialog" code={`<div class="dialog-backdrop dialog-backdrop-show">
  <!-- dialog content -->
</div>`} />

## JavaScript Integration

Here's a simple example of showing and hiding dialogs:

<ComponentShowcase title="JavaScript Integration" code={`<button class="btn btn-primary" id="openDialog">Open Dialog</button>

<div class="dialog-backdrop" id="myDialog">
  <div class="dialog">
    <div class="dialog-header">
      <h2 class="dialog-title">Example Dialog</h2>
      <button class="dialog-close" id="closeDialog" aria-label="Close">×</button>
    </div>
    <div class="dialog-body">
      <p>Dialog content goes here.</p>
    </div>
    <div class="dialog-actions">
      <button class="btn btn-text" id="cancelDialog">Cancel</button>
      <button class="btn btn-primary" id="confirmDialog">Confirm</button>
    </div>
  </div>
</div>

<script>
  const dialog = document.getElementById('myDialog');
  const openBtn = document.getElementById('openDialog');
  const closeBtn = document.getElementById('closeDialog');
  const cancelBtn = document.getElementById('cancelDialog');

  // Open dialog
  openBtn.addEventListener('click', () => {
    dialog.classList.add('dialog-backdrop-show');
  });

  // Close dialog
  const closeDialog = () => {
    dialog.classList.remove('dialog-backdrop-show');
  };

  closeBtn.addEventListener('click', closeDialog);
  cancelBtn.addEventListener('click', closeDialog);

  // Close on backdrop click
  dialog.addEventListener('click', (e) => {
    if (e.target === dialog) {
      closeDialog();
    }
  });

  // Close on escape key
  document.addEventListener('keydown', (e) => {
    if (e.key === 'Escape' && dialog.classList.contains('dialog-backdrop-show')) {
      closeDialog();
    }
  });
</script>`} />

## Best Practices

### Accessibility

- Use semantic HTML and ARIA attributes
- Trap focus within the dialog when open
- Return focus to trigger element when closed
- Support keyboard navigation (Escape to close)
- Provide clear labels for close buttons

<ComponentShowcase title="Accessible Dialog" code={`<div class="dialog-backdrop" role="dialog" aria-modal="true" aria-labelledby="dialogTitle">
  <div class="dialog">
    <div class="dialog-header">
      <h2 class="dialog-title" id="dialogTitle">Accessible Dialog</h2>
      <button class="dialog-close" aria-label="Close dialog">×</button>
    </div>
    <div class="dialog-body">
      <p>This dialog follows accessibility best practices.</p>
    </div>
    <div class="dialog-actions">
      <button class="btn btn-text">Cancel</button>
      <button class="btn btn-primary">Confirm</button>
    </div>
  </div>
</div>`} />

### User Experience

1. **Use dialogs sparingly** - Don't interrupt the user unnecessarily
2. **Keep content focused** - Present one task or decision at a time
3. **Provide clear actions** - Use descriptive button labels
4. **Enable easy dismissal** - Support Escape key and backdrop clicks
5. **Consider mobile** - Use fullscreen or full variants on small screens

## Framework Examples

### React

```tsx
import { useState } from 'react';

interface DialogProps {
  isOpen: boolean;
  onClose: () => void;
  title: string;
  children: React.ReactNode;
  size?: 'sm' | 'md' | 'lg' | 'xl' | 'full' | 'fullscreen';
}

export function Dialog({ isOpen, onClose, title, children, size = 'md' }: DialogProps) {
  return (
    <div
      className={`dialog-backdrop ${isOpen ? 'dialog-backdrop-show' : ''}`}
      onClick={(e) => e.target === e.currentTarget && onClose()}
    >
      <div className={`dialog dialog-${size}`}>
        <div className="dialog-header">
          <h2 className="dialog-title">{title}</h2>
          <button className="dialog-close" onClick={onClose} aria-label="Close">
            ×
          </button>
        </div>
        <div className="dialog-body">
          {children}
        </div>
      </div>
    </div>
  );
}

// Usage
function App() {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <>
      <button className="btn btn-primary" onClick={() => setIsOpen(true)}>
        Open Dialog
      </button>

      <Dialog
        isOpen={isOpen}
        onClose={() => setIsOpen(false)}
        title="Example Dialog"
        size="md"
      >
        <p>Dialog content goes here.</p>
      </Dialog>
    </>
  );
}
```

### Vue

```vue
<template>
  <div>
    <button class="btn btn-primary" @click="isOpen = true">
      Open Dialog
    </button>

    <div
      :class="['dialog-backdrop', { 'dialog-backdrop-show': isOpen }]"
      @click.self="isOpen = false"
    >
      <div :class="`dialog dialog-${size}`">
        <div class="dialog-header">
          <h2 class="dialog-title">{{ title }}</h2>
          <button class="dialog-close" @click="isOpen = false" aria-label="Close">
            ×
          </button>
        </div>
        <div class="dialog-body">
          <slot />
        </div>
      </div>
    </div>
  </div>
</template>

<script setup>
import { ref } from 'vue';

defineProps({
  title: {
    type: String,
    required: true
  },
  size: {
    type: String,
    default: 'md'
  }
});

const isOpen = ref(false);
</script>

<!-- Usage -->
<Dialog title="Example Dialog" size="md">
  <p>Dialog content goes here.</p>
</Dialog>
```

## API Reference

### Class Names

| Class | Description |
|-------|-------------|
| `.dialog-backdrop` | Backdrop/overlay container (required) |
| `.dialog-backdrop-show` | Shows the dialog with animation |
| `.dialog` | Dialog container (required) |
| `.dialog-header` | Header section |
| `.dialog-title` | Dialog title text |
| `.dialog-close` | Close button |
| `.dialog-body` | Content section |
| `.dialog-actions` | Footer with action buttons |
| `.dialog-sm` | Small dialog (20rem max-width) |
| `.dialog-md` | Medium dialog (28rem max-width, default) |
| `.dialog-lg` | Large dialog (36rem max-width) |
| `.dialog-xl` | Extra large dialog (48rem max-width) |
| `.dialog-full` | Full viewport dialog with padding |
| `.dialog-fullscreen` | Fullscreen dialog (no padding/radius) |
| `.dialog-alert` | Alert/confirmation dialog variant |
| `.dialog-center` | Center-align all content |
| `.dialog-icon` | Icon container |
| `.dialog-icon-primary` | Primary color icon |
| `.dialog-icon-secondary` | Secondary color icon |
| `.dialog-icon-tertiary` | Tertiary color icon |
| `.dialog-icon-success` | Success semantic icon |
| `.dialog-icon-error` | Error semantic icon |
| `.dialog-icon-warning` | Warning semantic icon |
| `.dialog-icon-info` | Info semantic icon |
| `.dialog-actions-start` | Left-align actions |
| `.dialog-actions-center` | Center-align actions |
| `.dialog-actions-between` | Space-between actions |
| `.dialog-scrollable` | Enable body scrolling (20rem max-height) |
| `.dialog-no-padding` | Remove body padding |
