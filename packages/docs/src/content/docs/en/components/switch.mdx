# Switch

Switches toggle the state of a single item on or off. @duskmoon-dev/core provides a Material Design 3 switch component with smooth animations and multiple variants.

## Basic Usage

```html
<label class="switch">
  <input type="checkbox" class="switch-input">
  <span class="switch-track">
    <span class="switch-thumb"></span>
  </span>
</label>
```

## With Label

Add descriptive labels to provide context for the switch:


## Best Practices

### When to Use

Switches are best for:

- **Instant changes**: Settings that take effect immediately
- **On/off states**: Binary choices like enable/disable features
- **System settings**: Preferences and configurations

```html
<div class="space-y-4">
  <label class="switch">
    <input type="checkbox" class="switch-input" checked>
    <span class="switch-track switch-primary">
      <span class="switch-thumb"></span>
    </span>
    <span class="switch-label">Enable notifications</span>
  </label>

  <label class="switch">
    <input type="checkbox" class="switch-input">
    <span class="switch-track">
      <span class="switch-thumb"></span>
    </span>
    <span class="switch-label">Auto-update apps</span>
  </label>

  <label class="switch">
    <input type="checkbox" class="switch-input" checked>
    <span class="switch-track switch-primary">
      <span class="switch-thumb"></span>
    </span>
    <span class="switch-label">Dark mode</span>
  </label>
</div>
```

### When Not to Use

Avoid switches for:

- **Form submission**: Use checkboxes instead when changes require user confirmation
- **Multiple options**: Use radio buttons or select dropdowns
- **Actions with delay**: If the change requires processing time, consider a button

### Accessibility

- Always use a `<label>` element for proper click area
- Provide descriptive label text that explains what the switch controls
- Use the native `<input type="checkbox">` for keyboard support
- Ensure sufficient color contrast in both states
- Support keyboard navigation (built-in with native input)

```html
<!-- Good: Accessible switch with clear label -->
<label class="switch">
  <input type="checkbox" class="switch-input" aria-describedby="wifi-description">
  <span class="switch-track">
    <span class="switch-thumb"></span>
  </span>
  <span class="switch-label">Wi-Fi</span>
</label>
<p id="wifi-description" class="text-sm text-gray-600">
  Connect to available wireless networks
</p>
```

### Visual Feedback

Provide clear visual feedback for all states:

1. **Default**: Clear unchecked appearance
2. **Hover**: Subtle background change
3. **Focus**: Visible outline for keyboard users
4. **Checked**: Distinct color and thumb position
5. **Disabled**: Reduced opacity

## Framework Examples

### React

```tsx
interface SwitchProps {
  label: string;
  checked?: boolean;
  disabled?: boolean;
  variant?: 'primary' | 'secondary' | 'tertiary';
  size?: 'sm' | 'md' | 'lg';
  onChange?: (checked: boolean) => void;
}

export function Switch({
  label,
  checked = false,
  disabled = false,
  variant = 'primary',
  size = 'md',
  onChange
}: SwitchProps) {
  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    onChange?.(e.target.checked);
  };

  const sizeClass = size !== 'md' ? `switch-${size}` : '';
  const variantClass = variant !== 'primary' ? `switch-${variant}` : 'switch-primary';

  return (
    <label className={`switch ${sizeClass}`.trim()}>
      <input
        type="checkbox"
        className="switch-input"
        checked={checked}
        disabled={disabled}
        onChange={handleChange}
      />
      <span className={`switch-track ${variantClass}`}>
        <span className="switch-thumb"></span>
      </span>
      <span className="switch-label">{label}</span>
    </label>
  );
}

// Usage
<Switch
  label="Enable notifications"
  checked={isEnabled}
  onChange={setIsEnabled}
/>
```

### Vue

```vue
<template>
  <label :class="switchClasses">
    <input
      type="checkbox"
      class="switch-input"
      :checked="modelValue"
      :disabled="disabled"
      @change="handleChange"
    />
    <span :class="trackClasses">
      <span class="switch-thumb"></span>
    </span>
    <span class="switch-label">{{ label }}</span>
  </label>
</template>

<script setup>
import { computed } from 'vue';

const props = defineProps({
  label: String,
  modelValue: Boolean,
  disabled: Boolean,
  variant: {
    type: String,
    default: 'primary'
  },
  size: {
    type: String,
    default: 'md'
  }
});

const emit = defineEmits(['update:modelValue']);

const switchClasses = computed(() => {
  const classes = ['switch'];
  if (props.size !== 'md') {
    classes.push(`switch-${props.size}`);
  }
  return classes.join(' ');
});

const trackClasses = computed(() => {
  const classes = ['switch-track'];
  if (props.variant !== 'primary') {
    classes.push(`switch-${props.variant}`);
  } else {
    classes.push('switch-primary');
  }
  return classes.join(' ');
});

const handleChange = (e) => {
  emit('update:modelValue', e.target.checked);
};
</script>

<!-- Usage -->
<Switch
  label="Enable notifications"
  v-model="isEnabled"
/>
```

### Svelte

```svelte
<script>
  export let label = '';

  export let variant = 'primary';
```


## API Reference

### Class Names

| Class | Description |
|-------|-------------|
| `.switch` | Base switch container (required) |
| `.switch-input` | Hidden checkbox input (required) |
| `.switch-track` | Visual track background (required) |
| `.switch-thumb` | Sliding thumb element (required) |
| `.switch-primary` | Primary color variant (default) |
| `.switch-secondary` | Secondary color variant |
| `.switch-tertiary` | Tertiary color variant |
| `.switch-sm` | Small size variant |
| `.switch-lg` | Large size variant |
| `.switch-label` | Label text |
| `.switch-label-left` | Position label on the left |
| `.switch-loading` | Loading animation state |
| `.switch-thumb-icon` | Optional icon in thumb |

### HTML Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `checked` | boolean | Sets the switch to checked state |
| `disabled` | boolean | Disables the switch interaction |
| `name` | string | Form field name |
| `value` | string | Form field value |
| `aria-describedby` | string | ID of element describing the switch |
